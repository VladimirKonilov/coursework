# Курсовая по сетевым технологиям

Проект реализует клиент-серверное TCP-приложение на C++

- транспортный протокол: `TCP`;
- собственный протокол прикладного уровня;
- регистрация нового пользователя;
- аутентификация пользователя при подключении;
- вычисление обратного и дополнительного кода числа для заданной разрядности;
- журналирование операций на сервере;
- обработка ошибок сетевого взаимодействия;
- ограничение на 3 одновременно подключенных клиента.

## Состав проекта

- `src/server_main.cpp` и `src/server_app.*` — сервер.
- `src/client_main.cpp` и `src/client_lib.*` — клиент.
- `src/protocol.*` — прикладной протокол.
- `src/conversion.*` — вычисление обратного и дополнительного кода.
- `src/user_store.*` — хранение пользователей.
- `tests/smoke_test.cpp` — проверка работоспособности.
- `PROTOCOL.md` — описание прикладного протокола.
- `USER_GUIDE.md` — руководство пользователя.

## Сборка

Быстрый вариант:

```bash
make
```

или через CMake напрямую:

```bash
cmake -S . -B build
cmake --build build
```

## Запуск

Сервер:

```bash
./build/netcourse_server 9090 data/users.db logs/server.log
```

или коротко:

```bash
make server
```

Клиент:

```bash
./build/netcourse_client 127.0.0.1 9090
```

или коротко:

```bash
make client
```

## Проверка работоспособности

Быстрый вариант:

```bash
make test
```

или напрямую:

```bash
cd build
ctest --output-on-failure
```

## Как пользоваться

1. Соберите проект.
2. Запустите сервер.
3. Запустите один или несколько клиентов.
4. В клиенте выберите регистрацию или вход.
5. После успешной аутентификации введите число и разрядность.
6. Клиент выведет обратный и дополнительный код.
7. Повторяйте вычисления или завершите работу.

## Замечания

- Для отрицательных чисел разрядность должна позволять представить модуль числа в формате со знаковым битом.
- При попытке подключения четвертого клиента сервер возвращает прикладную ошибку `SERVER_BUSY`.
- Подключение подтверждается сервером сразу после установки TCP-соединения, поэтому лишний клиент отсекается немедленно.
- Журнал операций сохраняется в файл, переданный серверу третьим аргументом.
