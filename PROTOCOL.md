# Прикладной протокол

Для обмена между клиентом и сервером используется текстовый протокол поверх TCP.

## Формат сообщения

- одно сообщение — одна строка;
- поля разделяются символом `|`;
- окончание сообщения — `\n`.

## Команды клиента

`REGISTER|username|password`

- регистрация нового пользователя;
- после успешной регистрации пользователь сразу считается аутентифицированным.

`LOGIN|username|password`

- вход существующего пользователя.

`CALC|number|bits`

- запрос на вычисление обратного и дополнительного кода;
- доступен только после аутентификации.

`QUIT`

- завершение соединения.

## Ответы сервера

Сразу после успешного TCP-подключения сервер отправляет приветственное сообщение:

`OK|CONNECTED|Connection established`

Если лимит подключений исчерпан, сервер немедленно возвращает:

`ERROR|SERVER_BUSY|Maximum number of clients reached`

Успешный ответ:

`OK|code|message|field1|field2|...`

Ответ об ошибке:

`ERROR|code|message`

## Основные коды ответов

- `REGISTERED` — регистрация выполнена.
- `AUTHENTICATED` — вход выполнен.
- `RESULT` — вычисление выполнено.
- `BYE` — соединение закрывается.
- `REGISTER_FAILED` — ошибка регистрации.
- `AUTH_FAILED` — ошибка аутентификации.
- `NOT_AUTHENTICATED` — пользователь не вошел в систему.
- `CALC_FAILED` — ошибка вычисления.
- `BAD_REQUEST` — неверный формат сообщения.
- `UNKNOWN_COMMAND` — неизвестная команда.
- `SERVER_BUSY` — превышен лимит подключений.
