# Руководство пользователя

## Назначение программы

Приложение состоит из сервера и клиента. Клиент подключается к серверу по TCP, проходит регистрацию или аутентификацию и отправляет число с разрядностью. Сервер вычисляет:

- обратный код;
- дополнительный код.

## Подготовка

Соберите проект:

```bash
make
```

или:

```bash
cmake -S . -B build
cmake --build build
```

## Запуск сервера

```bash
./build/netcourse_server 9090 data/users.db logs/server.log
```

или:

```bash
make server
```

Параметры:

- `9090` — порт сервера;
- `data/users.db` — файл базы пользователей;
- `logs/server.log` — файл журнала операций.

После запуска сервер ожидает подключения клиентов.
При успешном подключении сервер сразу подтверждает соединение. Если лимит клиентов уже исчерпан, новый клиент сразу получит отказ.

## Запуск клиента

```bash
./build/netcourse_client 127.0.0.1 9090
```

или:

```bash
make client
```

Параметры:

- `127.0.0.1` — адрес сервера;
- `9090` — порт сервера.

## Порядок работы в клиенте

1. После подключения выберите:
   - `1` — регистрация;
   - `2` — вход;
   - `0` — выход.
2. Введите логин и пароль.
3. После успешной аутентификации введите:
   - целое число;
   - разрядность.
4. Клиент отобразит:
   - обратный код;
   - дополнительный код.
5. Выберите, нужно ли выполнить еще одно вычисление.

## Обработка ошибок

- Если пароль неверный, клиент выведет код ошибки `AUTH_FAILED`.
- Если пользователь уже существует, будет выведена ошибка `REGISTER_FAILED`.
- Если число не помещается в заданную разрядность, будет выведена ошибка `CALC_FAILED`.
- Если к серверу пытается подключиться четвертый клиент, сервер вернет `SERVER_BUSY`.

## Журналирование

Сервер записывает в лог:

- подключения и отключения клиентов;
- регистрацию;
- аутентификацию;
- результаты вычислений;
- ошибки.

## Проверка работоспособности

```bash
cd build
ctest --output-on-failure
```
